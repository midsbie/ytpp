* ytpp

Post-process YouTube downloads for TV compatibility.

Older smart TVs (e.g. LG C8) can't decode AV1 video or Opus audio, the codecs that YouTube increasingly defaults to.  =ytpp= re-encodes these streams to H.265/AC3 while copying anything already compatible, so files play without hassle over DLNA/USB.

** Usage

#+begin_example
ytpp [options] <file>
#+end_example

| Option            | Description                                                    |
|-------------------+----------------------------------------------------------------|
| =-q, --quality <n>= | Quality level: QP for VAAPI, CRF for software (default: 18) |
| =--sw=            | Force software encoding (libx265) instead of VAAPI            |
| =--no-delete=     | Keep the original file after conversion                        |
| =-h, --help=      | Show help                                                      |

A convenience wrapper is included for download-and-transcode in one step:

#+begin_example
ytdl <yt-dlp args>
#+end_example

** How it works

1. Probe the input for codec and dimension info.
2. Re-encode heuristics:
   - If AV1: re-encode video to H.265.
   - If Opus: re-encode audio to AC3 (Dolby Digital).
   - Otherwise: streams are considered compatible and copied as-is.
3. VAAPI hardware encoding is used automatically when available, with a software (libx265) fallback.

*** VAAPI 32-byte alignment workaround

AMD and Intel VAAPI HEVC encoders (some?) (appear to) require frame heights divisible by 32.  Non-aligned heights (e.g. 1080) are internally padded to the next aligned value (1088), producing a green bar at the bottom of the frame.

=ytpp= works around this by scaling *up* to the aligned height in software, uploading to VAAPI, then scaling back *down* to the original height with =scale_vaapi=.  When the height is already aligned (720p, 2160p) the extra scaling is skipped entirely.

** Cookie support

YouTube's anti-bot measures often require browser cookies for yt-dlp to work reliably.  =ytcks= extracts cookies into a file that =ytdl= picks up automatically:

#+begin_example
ytcks [browser]
#+end_example

When called without an argument, =ytcks= makes a low-effort attempt to detect the default browser via =xdg-settings=.  Firefox users with multiple profiles may need to specify the profile path explicitly, since yt-dlp can pick the wrong one:

#+begin_example
ytcks "firefox:~/.mozilla/firefox/xxxxxxxx.profile-name"
#+end_example

This creates =cookies.txt= (=chmod 600=) in the current directory.  Subsequent =ytdl= invocations will pass =--cookies= to yt-dlp when =cookies.txt= exists in the working directory.  Set the =COOKIES= environment variable to override the path:

#+begin_example
COOKIES=~/cookies.txt ytcks firefox
COOKIES=~/cookies.txt ytdl <yt-dlp args>
#+end_example

** Dependencies

- =ffmpeg= / =ffprobe=
- =yt-dlp= (for =ytdl= only)
